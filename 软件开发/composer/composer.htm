有什么用

composer 解决的是项目与项目之间的相互依赖关系（或者可以理解为继承）。

比如说

             插件型项目A           负责把php数据套在html上(smarty)

             框架型项目B           比如thinkphp3.2（直接引入在composer.json中引入smarty的版本，无需手动拷贝）

             实际项目c               开发一个商城。我在composer.json中引入框架B即可，无需去引入项目A。但是项目A的代码最终必须下载到我的本地，谁去负责下载和管理？就是composer。解决和管理项目之间集成或引入的问题。

 

 

Composer 将这样为你解决问题：

a) 你有一个项目依赖于若干个库。

b) 其中一些库依赖于其他库。

c) 你声明你所依赖的东西。

d) Composer 会找出哪个版本的包需要安装，并安装它们（将它们下载到你的项目中）。

 

安装                     https://docs.phpcomposer.com/00-intro.html#Installation-Windows

                           记住安装后记得重启电脑

 

 

composer.json文件

 

{
    "require": {
        "monolog/monolog": "1.0.*"
    }
}

存放的位置     vendor/monolog/monolog/...

composer.lock文件

 记录当前已安装的全部依赖项目和其版本号。为了让代码在多个环境下，版本一致，把composer.json和composer.lock一起提交。

自动加载           

require 'vendor/autoload.php';

你可以在 composer.json 的 autoload 字段中增加自己的 autoloader。

{
    "autoload": {
        "psr-4": {"Acme\\": "src/"}
    }
}

Composer 将注册一个 PSR-4 autoloader 到 Acme 命名空间。

版本写法

 

名称 实例 描述
确切的版本号 1.0.2 你可以指定包的确切版本。
范围 >=1.0 >=1.0,<2.0 >=1.0,<1.1|>=1.2 通过使用比较操作符可以指定有效的版本范围。
有效的运算符：>、>=、<、<=、!=。
你可以定义多个范围，用逗号隔开，这将被视为一个逻辑AND处理。一个管道符号|将作为逻辑OR处理。
AND 的优先级高于 OR。
通配符 1.0.* 你可以使用通配符*来指定一种模式。1.0.*与>=1.0,<1.1是等效的。
赋值运算符 ~1.2 这对于遵循语义化版本号的项目非常有用。~1.2相当于>=1.2,<2.0。想要了解更多，请阅读下一小节。

更新镜像

composer config -g repo.packagist composer https://packagist.phpcomposer.com

取消镜像使用默认镜像：

composer config -g --unset repos.packagist
